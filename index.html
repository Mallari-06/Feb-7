Correlation

Correlation is the process of relating resources to each other. Correlation automatically identifies the relationships between the resources that are managed by MultiSystem Manager topology features and SNA topology manager, and are discovered by different agents. To enable the correlation function, include the FLCSDM8 RODM load file in your structure load by uncommenting it in the CNMSJH12 job.

Correlation dynamically associates cross-protocol resources with the system in which they are contained. Because this facility continuously updates topology and status, operators can isolate the source of a problem reported by agents. For example, an operator can take the following kinds of actions:

• Isolate whether the failure between 2 z/OS system images was caused by a TCP/IP stack failure as reported by the IBM Tivoli Network Manager feature.
• Respond to a workstation failure report by using one view to show the status of all the components of the workstation.

The correlation function is implemented using RODM methods. Correlation is performed on either a network address or a free-form string:

• LAN MAC address (uses the aIndMACAddress field)
• IP address (uses the iPAddress field)
• Free-form, user-provided string (uses the Correlater field)

Network address correlation updates an object of the aggregateSystem class (or creates one if necessary) in RODM to contain all managed resources that are correlated.

Free-form correlation updates an object of the GMFHS_Aggregate_Objects_Class class (or creates one if necessary) in RODM to contain all objects that are correlated. You can enter a multi-value string in the Correlater field to create or locate a hierarchy of correlated objects. For example, you can enter ‘Rm12 Building3 Montreal Canada’.

You can also correlate objects that you have created. Refer to the IBM Tivoli NetView for z/OS Resource Object Data Manager and GMFHS Programmer's Guidefor more information.

See the IBM Tivoli NetView for z/OS User's Guide: NetView Management Console for more information about how operators can use the correlation function.

This class is used as a computer system that runs a distributed operating system such as Windows, Linux, or Unix.

ZOS class is a child of OperatingSystem, with the same behavior and the same base set of attributes.

The attributes identified in View navigation for this resource are taken from a combination of the logical operating system instance (CDM class OperatingSystem) and the physical machine instance (CDM class ComputerSystem). MSM discovers software and networking aspects of a distributed system. If the hardware Model No, Serial number, or physical location can be identified, ComputerSystem class would be created. MSM does not discover that information.

Combining physical and logical system attributes and behaviors into one class is similar to the System representation done by IBM Tivoli Network Manager. Network Manager identifies logical and physical resources in the combination of Chassis and Entity classes.
